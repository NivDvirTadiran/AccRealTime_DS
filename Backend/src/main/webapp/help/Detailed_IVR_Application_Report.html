Detailed IVR Application Report

Note: The  icon denotes that this particular function is not available in Aeonix Contact Center Lite.

The Detailed IVR Application Report shows detailed information concerning one particular IVR Application. The information is displayed in a form window.

To create a Detailed IVR Application Report:

  1. Click  (Open Report or Template) in the Control Area.

  1. In the Open dialog box, select IVR Applications from the Template tab.

  1. Select the Detailed Report template. The Select IVR App dialog box is displayed.

  2. Select the IVR application for which you want to obtain detailed status information.

  1. You can save the report by selecting (Save Report As) in the Control Area.

The Detailed IVR Application Report contains the following fields:

On Line

The On Line section shows the current activity handled by the IVR application.

Inbound

Calls

Number of incoming calls currently handled by the IVR application

Too Long

Number of incoming calls currently handled by the IVR application with handling time that is longer than a predefined threshold.

Avg.time

Average handling time of the incoming calls that are currently in the IVR application

Max time

Longest handling time of the incoming calls that are currently in the IVR application

Outbound

Calls

Number of outbound calls currently handled by the IVR application

Too Long

Number of outbound calls currently handled by the IVR application with handling time that is longer than a predefined threshold.

Avg.time

Average handling time of the outbound calls that are currently in the IVR application

Max time

Longest handling time of the outbound calls that are currently in the IVR application

On Current Interval

The On Current Interval section gives statistical information about inbound and outbound calls handled by the IVR application on this interval.

Inbound

The Inbound section gives statistical information about inbound calls

ACD calls

RPH

Number of accesses of incoming calls per hour

All/access

Number of accesses of incoming calls in the current interval

Fully handled

The number of accesses of incoming calls that were fully handled. A fully handled access is a call that went through the end point of the application.

Failed

The number of accesses of incoming ACD calls that failed. A failed access is a call that hung up before the end point of the application.

Interflowed

The number of accesses of incoming ACD calls that interflowed. An interflowed access is a call that while processed, routed to a destination that is not part of the application (like an agent or a different script that is not part of the application).

Avg.time (sec)

All/access

The average time that a ‘call access’ is in the IVR application

Fully handled

The average time that a fully handled ‘call access’ is in the IVR application. A fully handled access is a call that went through the end point of the application

Failed

The average time that a failed ‘call access’ is in the IVR application. A failed access is a call that hang up before it went through the end point of the application.

Interflowed

The average time that a failed ‘call access’ is in the IVR application. An interflowed access is a call that while processed, routed to a destination that is not part of the application (like an agent or a different script which is not part of the application).

Max time (sec)

All/access

The longest time that a ‘call access’ is in the IVR application

Fully handled

Failed
