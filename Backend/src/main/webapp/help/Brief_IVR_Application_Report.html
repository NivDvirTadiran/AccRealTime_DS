Brief IVR Application Report

Note: The  icon denotes that this particular function is not available in Aeonix Contact Center Lite.

The Brief IVR Application Report shows information on one or more IVR Applications for the current interval.

To create a Brief IVR Application Report window:

  1. Click  (Open Report or Template) in the Control Area.

  1. In the Open dialog box, select IVR Applications from the Template tab.

  1. Select the Brief Report template. The Select Multiple IVR App dialog box is displayed.

  2. Select the IVR applications for which you want to obtain the status, and then click Select.

  1. You can save the report by selecting (Save Report As) in the Control Area.

The following columns are displayed in the Brief IVR Application Report:

IVR App Name

The name of the IVR Application

Calls ACD

Counts the number of incoming ACD calls that are currently being processed by the IVR application.

Too Long ACD

Counts the number of incoming ACD calls that are currently being processed by the IVR application for more than the predefined threshold.

Times Accessed ACD

Counts the number of times that incoming ACD calls accessed the IVR application. A call that accessed the same IVR application more than once, will be counted more than once.

Fully Handled ACD

The number of accesses of incoming ACD calls that were fully handled. A fully handled access is a call that went through the end point of the application.

Failed ACD

The number of accesses of incoming ACD calls that failed. A failed access is a call that hung up before connecting with end point of the application.

Interflowed ACD

The number of accesses of incoming ACD calls that interflowed. An interflowed access is a call that while processed, routed to a destination that is not part of the application (like an agent or a different script which is not part of the application).

RPH ACD

The number of fully handled accesses incoming calls per hour.

Calls OACD

Counts the number of outbound ACD calls that are currently being processed by the IVR application.

Too Long OACD

Counts the number of outbound ACD calls that are currently being processed by the IVR application for more than the predefined threshold.

Times Accessed OACD

Counts the number of times that outbound ACD calls accessed the IVR application. A call that accessed the same IVR application more than once, will be counted more than once.

Fully Handled OACD

The number of accesses of outbound ACD calls that were fully handled. A fully handled access is a call that went through the end point of the application.

Failed OACD

The number of accesses of outbound ACD calls that failed. A failed access is a call that hang up before the end point of the application.

Interflowed OACD

The number of accesses of outbound ACD calls that interflowed. An interflowed access is a call that while processed, routed to a destination that is not part of the application (like an agent or a different script which is not part of the application).

RPH OACD

The number of fully handled accesses of outbound calls per hour.
