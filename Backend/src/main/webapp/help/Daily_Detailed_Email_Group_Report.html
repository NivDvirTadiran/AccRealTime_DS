Daily Detailed Email Group Report

Note: The  icon denotes that this particular function is not available in Aeonix Contact Center Lite.

The Daily Detailed Email Group Report shows detailed information concerning one particular email group. The information is displayed in a form window. This report presents the daily data accumulated since midnight.

To create a Daily Detailed Email Group Report:

  1. Click  (Open Report or Template) in the Control Area.

  1. In the Open dialog box, select Groups from the Template tab.

  1. Select the Email Daily Detailed Report template. The Select Email Group dialog box is displayed.

  2. Select the group for which you want to obtain detailed status information.

Note: You can change the group presented in the report by selecting the current group in the top bar of the window, and then selecting the required group from the Select Email Group dialog box.

  1. You can save the report by selecting (Save Report As) in the Control Area.

Note: The colors in some fields indicate that the thresholds for those fields have been exceeded. Thresholds are specified in the Aeonix Contact Center Administration.

The Daily Detailed Email Group Report contains the following fields:

Contacts in queue

This section gives information about the emails that are currently in the groupâ€™s queue

Contacts

The number of currently queued email contacts

Avg.time

The average wait time in queue of all currently queued email contacts

Above T.ASA

The number of email contacts that are currently queued for more then the predefined target ASA.

Max time

The queue time of the email contact that is currently waiting in queue the longest.

Agents

This section gives information about the number of agents in the specified state

Logged in

The number of currently logged in agents

Release

The number of agents that are currently in a release state (includes all release codes). An agent that while in release state is busy with a non-ACD call, will be counted as released.

Wrap-up

The number of agents that are currently in a wrap-up state

Busy

The number of agents that are currently in a busy state

Idle

The number of agents that are currently in an idle state

ACD

The number of agents that are currently in an ACD state (includes all media types, i.e., voice, chat and email)

Non-ACD

The number of agents that are currently in a non-ACD  state (includes incoming outgoing, internal and external)

Split

The number of agents that are currently handling an ACD contact in another group (includes all media types, i.e., voice, chat and email). The Split state starts when the call starts to ring and ends after the wrap-up.

Current Contacts Talk Time

This section gives information about email contacts that are currently handled by agents of the group.

# too long

The number of email contaats that are handled by an agents and their interaction time exceeds the predefined threshold.

Max

The longest interaction time of currently handled email contact

On Current Day

This section gives statistical information about the current day. The statistics are accumulated since last midnight.

ACD contacts

Accptd

Number of email contacts accepted by the group

Ansd

Number of email contacts answered by the group

Intfl.out

Number of email contacts accepted by the group and interflowed.

Avg.time(sec)

In Q

The average wait time in queue of email contacts that waited in the queue since last midnight

Talk

The average interaction time of email contacts since last midnight

Intfl

The average wait time in queue of email contacts that waited in the queue and eventually interflowed out since last midnight

Max.time(sec)

In Q

The longest wait time in queue of email contacts that waited in the queue since last midnight

Talk

The longest interaction time of email contacts since last midnight

Intfl

The longest wait time in queue of email contacts that waited in the queue and eventually interflowed out since last midnight

Statistics

TSF

The TSF is a parameter that measures the level of service given by the group. It is calculated by dividing the number of contacts answered before the target ASA by accepted contacts.

Note: There are two other ways to calculate the TSF. The first is to divide it by the number of answered contacts. The second is to divide it by the sum of answered and abandoned after target ASA contacts. The selection between the options is done in the administration and the selected option is applicable to all groups and DNIS numbers.

RPH

The number of answered email contacts in an hour

ASA

The average speed of answer.
